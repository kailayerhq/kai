Kai

Intent-based version control. Track what you meant to do, not just what changed.

View on GitLab Quick Start

Why Kai?

Git commit

fix: update validation

Kai intent

Add formatDate and formatTime in Utils

Kai automatically analyzes your code changes and generates meaningful descriptions. No more guessing what a commit did from a vague message.

Features

Semantic Change Detection

Detects function additions, API changes, condition modifications, and more using AST analysis.

Auto-Generated Intents

Creates human-readable descriptions like "Add formatDate and formatTime in Utils" automatically.

Module Mapping

Map file paths to logical modules (Auth, API, Frontend) for better organization.

Content-Addressed Storage

Files stored by BLAKE3 hash. Efficient, deduplicated, and verifiable.

Quick Start

# Initialize a kai repository
kai init

# Create a snapshot of your code
kai snapshot -m "Initial version"

# Create and start a workspace
kai ws create main
kai ws start main

# Make changes to your code, then stage
kai ws stage main -m "Add new features"

# View the auto-generated intent
kai intent render <changeset-id>
Intent: Add formatDate and formatTime in Utils

Commands

kai init                 # Initialize repository
kai snapshot             # Create a snapshot
kai status               # Show changed files
kai diff                 # Show file differences
kai ws create <name>     # Create workspace
kai ws stage <name>      # Stage changes
kai changeset list       # List changesets
kai intent render <id>   # Show intent for changeset
kai log                  # View history
kai checkout <ref>       # Restore a snapshot
